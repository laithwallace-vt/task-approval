Slot 1 — Header (2 sub-slots)
1a Chrome: Task type label (left) + entity breadcrumb with icon + close. Panel toggle (💬) visible as icon — no label.
1b Meta: Assignee with avatar + pencil edit affordance. Due date. Omit if task type doesn't configure these.
Frame name: task/ratification/header
2
Slot 2 — Body: single_view layout
Previous design: radio buttons for Approve/Deny — confuses mobile users, looks like a form, not a decision.
Replacement: two full-width decision cards with icon + label. Selected state uses coloured border + fill. Matches Linear/Asana "status select" pattern.
Vendor hero card: Always full-width. No competitor comparison needed here (that's War Room).
AI banner: Shows when Vertice AI has a recommendation. Can be positive (recommend) or cautionary (flag risk). Hidden if no signal.
Notes: Optional, never required for Ratification. Placeholder copy: "Add any notes…"
3
Slot 3 — Panel: collapsed default
Previous: details squeezed in a narrow sidebar — not enough room, creates visual noise.
Panel collapsed by default for Ratification (most approvals need no context). Toggle via 💬 icon in header. When expanded: 240px wide, tabs for Comments / Details.
panelSections: ["details", "comments"] — Details tab first (most useful for quick context).
4
Slot 4 — Actions: task mode
Approve button: Enabled only when a decision is selected. Disabled + greyed if neither option selected.
Decline button: Left-anchored. Destructive styling. Opens confirmation if notes field is empty.
No "Cancel" or "Back" — Ratification is a single-step task. Close × serves as escape.
5
CRISP check
C
Task type label + entity breadcrumb gives instant orientation. User knows what they're approving.
R
AI recommendation visible immediately. Budget check inline. No navigation required.
I
AI explains its recommendation. Budget validation shown. Line items available on demand.
S
Decision buttons replace radio + form pattern. One click to select, one to confirm. No page navigation.
P
Panel available for deep-dive. Request details accordion for full context. Notes optional, never blocking.

Current issues (DES-627)
No action buttons — user must navigate to platform to approve anything.
Not enough detail in the email to approve confidently without opening the task.
✓
New email spec
Subject line pattern: "Action required: [task type] [vendor] — [amount]". Scannable in inbox.
Vendor card: Logo, total, savings delta, billing cadence, requester, cost centre, budget status.
AI recommendation: 1 sentence. Positive or cautionary. Always present if AI signal available.
Action buttons: Magic link tokens. Decision recorded without login for low-risk Ratification. High-risk tasks require login confirmation.
Fallback link: "View full request in Vertice" for users who want more context before deciding.
⚙
Engineering notes
Approve/Decline buttons are signed, time-limited URLs (?token=xxx&action=approve).
On click: server validates token, records decision, redirects to confirmation page. No login required for standard approvals.
For high-value tasks (>£100k threshold, configurable): token redirects to platform login then confirms decision. Not inline.
Email rendered as HTML. Tested in Gmail, Outlook (desktop + web), Apple Mail.

Current issues (DES-627)
Custom tasks have no approval buttons in Slack — only "View in platform".
Insufficient context. User can't make a decision from the notification alone.
✓
New Slack Block Kit spec
Message text: Personalised ("👋 Alex Chen") with task type + due date. Friendly, not robotic.
Attachment card: Left border = Vertice purple. Task name + type as title.
Fields: Amount, savings (colour-coded), requester, due date. 4 fields maximum — no noise.
AI strip: 1 sentence recommendation inline. Shown only when signal available.
Buttons: Approve + Decline interactive (Block Kit actions). "View details" as tertiary. After action: message updates to show decision made + timestamp.
⚙
Post-action message update
On Approve: message updates — buttons replaced with "✓ Approved by Alex Chen · 9:31 AM". No re-message needed.
On Decline: message updates — "✗ Declined by Alex Chen · reason: [notes if added]".
Teams equivalent: Adaptive Card with same fields + actions.
