Meta placement — density rule
body_density set explicitly by form author. Never inferred from zone count.
"sparse"
1–2 zones. Meta in body (slot 2a). Decision-focused.
"dense"
3+ zones. Meta in header (slot 1a). Body needs height.

Key rules
🔒Shell HTML never changes between task types. All variation is in the definition object.
↕Zones render in array order. No layout logic inside the shell.
📋Comments is just a section type. Not a named concept the shell knows about.
🎯Buttons positioned by role, not by task type. Secondary left, rest right.
⚡Urgency override: dueDate ≤ 24h always pushes meta to header regardless of density.
