05 · Notification Channels — Current issues + solutions (DES-627)
📧
Email
2 critical issues
needs design
No action buttons. User must navigate to platform. Breaks the zero-friction approval for executives.
Insufficient context. Not enough detail to approve without opening the task.
✓ Solution
Rich HTML email with: vendor logo, amount, savings delta, AI recommendation badge. Inline Approve / Decline buttons (magic link tokens). Decision recorded without login for low-risk Ratification tasks.
#
Slack / Teams
2 issues
needs design
Custom tasks missing approval buttons. Only "View in platform" available.
Insufficient context similarly to email — lacking details to decide.
✓ Solution
Slack Block Kit card: task name, vendor, amount, savings pill, AI recommendation. Interactive buttons: Approve / Decline / View details. Response updates message state (no re-open needed).
🔗
In-platform (Task link)
1 issue
Task entry point unclear. Link is in active tasks section and requires clicking a simplified workflow card — not obvious for occasional approvers.
✓ Solution
Persistent "Pending your action" banner at top of dashboard. Direct deep-link from any notification. Task counter badge on nav. Request details page: sticky "Approve now" CTA when task is pending current user.
06 · Archetype Decision Matrix — for engineering and design team
Scenario Archetype body_layout actions_mode Panel default Left panel Mobile tab Principle
User confirms one thing (renewal, simple approval) Ratification single_view task Collapsed ✗ Task tab Velocity over process
User chooses between multiple offers / vendors War Room comparison_grid task Expanded ✗ Split cards Show data delta, highlight winner
User enters data from a document (offer details) Input · Source split_source task Collapsed ✓ PDF Swipe left/right Show source, AI-prefill fields
User completes multi-step data collection Input · Stepped stepped wizard Collapsed ✓ Step nav Progress bar top One step at a time, clear progress
System is waiting on external (DocuSign, Ironclad, vendor) Pulse status_hero passive Hidden ✗ Status only Human language, not status codes
Shell contract: The shell renders 4 named slots. Task type definitions provide body_layout, panelSections[], and actions_mode. The left panel (PDF viewer, step nav) is always body-internal — the shell never renders it as a named slot. New task types are data configuration, not new screens.
