Task layout system — Assignee & Due Date placement

Where meta-information lives, and why — for alignment with Maryna

Header placement Dense tasks (3+ content zones). User is in form-filling mode. Body space is valuable — don't waste it on meta.
Body placement Sparse tasks (1–2 content zones). User is in decision mode. Context and action are both visible at once.
Never duplicate Meta in both places creates confusion — which one is authoritative? Pick one location per task type. Always.
Override: urgency If the due date is overdue or ≤24h, always promote it to the header with a warning treatment — regardless of task type.

Maryna's heuristic — use this to decide without asking
Count the distinct content zones in the body. If there are 3 or more (e.g. vendor card + request details + offer form), meta goes in the header. If there are 1–2, meta goes in the body. If due date is overdue or within 24 hours, always surface it in the header with warning colour — regardless of task type.
